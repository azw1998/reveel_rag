Document ID: surcharge_exact_matches_schema
Document Type: schema
Databricks Catalog: prod
Databricks Schema: surcharges
Table Name: exact_matches

Description:
A mapping table that identifies what surcharge_id and surcharge_name a particular surcharge should have given the values of the other columns.

Grain:
One row per (
    'cleaned_cd',
    'is_alaska',
    'is_extended',
    'is_hawaii',
    'is_hwt',
    'is_intra_hawaii',
    'is_remote',
    'is_residential',
    'receiver_country_territory',
    'receiver_state',
    'sender_country_territory',
    'sender_state',
    'is_transportation_charge',
    'charge_description_code',
    'service',
    'new_service_type',
    'import_export',
    'carrier',
).

Primary Use Cases:


Other notes:
is_hawaii, is_alaska, is_intra_hawaii are all mutually exclusive
Joins on each column should be null safe equals.

Columns:

1. cleaned_cd: String
Description: is the cleaned charge description of the surcharge as found in the invoice files. Usually trimmed and lowercased, so make sure joins onto this column are cased and cleaned.

2. is_alaska: Boolean
Description: Flags whether the charge is from an Alaska shipment.

3. is_extended: Boolean
Description: Flags whether the charge is to zip code that's considered extended.

4. is_hawaii: Boolean
Description: Flags whether the charge is from a Hawaii shipment.

5. is_hwt: Boolean
Description: Flags whether the charge is from a hundredweight shipment.

6. is_intra_hawaii: Boolean
Description: Flags whether the charge is from an intra-Hawaii shipment.

7. is_remote: Boolean
Description: Flags whether the charge is to zip code that's considered remote.

8. is_residential: Boolean
Description: Flags whether the charge is from a residential shipment.

9. receiver_country_territory: String
Description: receiver_country_territory

10. receiver_state: String
Description: receiver_state

11. sender_country_territory: String
Description: sender_country_territory

12. sender_state: String
Description: sender_state

13. is_transportation_charge: Boolean
Description: Flags whether the charge is from a transportation_charge. Only transportation charges should get surcharge_ids.

14. charge_description_code: String
Description: ups charges have charge_description_code. for all other carriers this field will be null. Just like with charge_description, make sure that joins using this column are trimmed and lowercased.

15. service: String
Description: the service of the charge.

16. new_service_type: String
Description: the normalized service of the charge. 

17. import_export: String
Description: whether the charge is from an import, export, domestic, or other shipment.

18. carrier: String
Description: the carrier that shipped the shipment that the charge is from.

19. surcharge_id: String
Description: the surcharge id

20. surcharge_name: String
Description: the normalized surcharge name associated with the surcharge id
