Document ID: carrier_rates_schema
Document Type: schema
Databricks Catalog: prod
Databricks Schema: rates
Table Name: carrier


Description:
The carrier_rates table stores normalized shipping rate data by carrier, service level, shipping zone, and billable weight. Each row represents a single price point for a specific combination of carrier, service, zone, and weight.

Grain:
One row per (
 'carrier',
 'service',
 'weight',
 'zone',
 'import_export',
 'container_type',
 'start_date',
 'end_date',
 'is_hawaii',
 'is_intra_hawaii',
 'is_alaska',
 'is_intra_alaska'
).

Primary Use Cases:
- Retrieve price for a given carrier/service/zone/weight
- Compare service pricing within a carrier
- Identify cheapest service for a given weight and zone
- Analyze price differences across zones

Columns:

3. zone (STRING)
   Description: Carrier-defined geographic pricing zone.
   Example values: 102, 103, 104, 105, etc.
   These are exact matches.

4. weight (INT)
   Description: Billable shipment weight in pounds.
   If the user asks for a weight less than 1, filter for weight = 0.
   Example values: 0, 1, 2, 5, 10, 25, 50


5. cost_curr (DECIMAL)
   Description: Published transportation rate in USD.
   Usually users will ask for the rate, so this is what we want to return.

6. import_export (STRING)
   Description: States whether the rate is for an imported shipments or exported shipments.
   Distinct values are "import", "export", or null.
   Some services have a different cost for import or export. Unless the user specifies the type of import_export, you should query for import_export is null.
   If the service has "import" or "export" in the name but the user doesn't specify the type of import_export, filter for the type of import_export or null.
   e.g. If they ask about international economy export but don't ask for import_export, filter import_export for import_export is null or import_export = 'export'

7. container_type (STRING)
   Description: Some services have different costs for different container types.
   Distinct values:
    10 KG
    25 KG
    envelope
    FedEx 10Kg Box
    FedEx 25Kg Box
    pak
   Unless the user specifies the type of container type, you should query for container_type is null.

8. start_date (DATE)
   Description: The start date of the when rate become effective.

9. end_date (DATE)
   Description: The end date of when the rate is in effect. 
   An end_date of null means that the rate is currently in effect.

10. is_hawaii (BOOLEAN)
   Description: Whether or not the rate is hawaii-specific. This is mutually exclusive from intra-hawaii.
   Unless the user specifies the type of is_hawaii, query for is_hawaii is False.

11. is_alaska (BOOLEAN)
   Description: Whether or not the rate is alaska-specific. This is mutually exclusive from intra-alaska.
   Unless the user specifies the type of is_alaska, query for is_alaska is False.

12. is_intra_hawaii (BOOLEAN)
   Description: Whether or not the rate is intra-hawaii-specific. This is mutually exclusive from hawaii.
   Unless the user specifies the type of is_intra_hawaii, query for is_intra_hawaii is False.

13. is_intra_alaska (BOOLEAN)
   Description: Whether or not the rate is intra-alaska-specific. This is mutually exclusive from alaska.
   Unless the user specifies the type of is_intra_alaska, query for is_intra_alaska is False.


Example Queries:

1. Question: 'What is the current rate for fedex 2 Day Standard 3 lbs and zone 4?':'
   Answer:
    '''select
    cost_curr
    from
    prod.rates.carrier
    where
    carrier = 'fedex'
    and service = '2Day'
    and weight = 3
    and zone = '4'
    and not is_hawaii
    and not is_alaska
    and not is_intra_alaska
    and not is_intra_hawaii
    and current_date() between start_date and coalesce(end_date, '2099-12-31')
    order by
    start_date desc
    limit 1'''

2. Question: 'What did ups ground comm hawaii rate cost for 3 lb and zone 44 on 2025-12-10?':
    Answer: 
    '''select
    cost_curr
    from
    prod.rates.carrier
    where
    carrier = 'ups'
    and service = 'Ground Commercial'
    and weight = 3
    and zone = '44'
    and is_hawaii
    and not is_alaska
    and not is_intra_alaska
    and not is_intra_hawaii
    and '2025-02-10' between start_date and coalesce(end_date, '2099-12-31')
    order by
    start_date desc'''

3. Question: 'What did fedex international priority import cost for zone PR and weight 2 between '2026-01-01' for container type pak?:
   Answer: '''
   select
  *
from
  prod.rates.carrier
where
  carrier = 'fedex'
  and service = 'IP-Imp'
  and weight = 2
  and zone = 'PR'
  and (import_export is null or import_export = 'import')
  and container_type rlike 'pak'
  and not is_hawaii
  and not is_alaska
  and not is_intra_alaska
  and not is_intra_hawaii
  and '2026-01-01' between start_date and coalesce(end_date, '2099-12-31')
order by
  start_date desc
limit 1
   '''
